What is Advent of Code?

Advent of Code is a unique set of coding challenges created by Eric Wastl released annually at the beginning of December. These challenges follow a holiday-themed storyline where each challenge ties into the next, with challenges growing in difficulty. Every individual participant is given a unique problem input for each day's challenge. Using any coding language of your choice, the objective is to complete all 25 days of challenges. To learn more, head over to the official Advent of Code website!

    Part 1 Solution

    In the first part of Day 1, I begin by importing the regex library. Regex is extremely helpful when searching, replacing, and extracting values in strings of text, and was my preferred tool in developing a solution to this problem. Once the document was imported and split into clean lines, all values which were not integers were stripped from the document, leaving only lines of numbers. Indexing each line, the first and last numbers were combined to create a single two-digit number. These numbers were placed in a list and added together resulting in the answer to the first part of Day 1.

    Part 2 Solution

    Part 2 raises a new issue with the calibration input document. It turns out there are actually numbers speeled out in each of the lines that should also be counted when developing the list of calibration numbers. Spellings like "seven" and "four" need to appear as "7" and "4". There is an additional layer of complexity though - there are numbers that are spelled using letters from other numbers, e.g. "eightwo" need to become "82" to correctly create the calibration number.

    Regex was again used. This time, once divided into lines, spellings for each number were located and replaced with their respective values using a dictionary of conversions. In order to account for the combined spellings, spellings were replaced with a number indexed into each spelling (i.e. "one" was converted to "o1e", "two" to "t2o" etc.). In practice, this first turns the above "eightwo" into "e8ttwo", which will then convert the string to "e8tt2o", allowing me to extract both "8" and "2" from the line.

    The rest of the steps mirror part 1, removing all letters, indexing the remaining numbers, adding them to the empty calibration numbers list, and summing their values for the answer.

Inferences can then be made on the number of cubes given the values in each pull. Your job for the first part of this day is to look at a list of game results and determine how many of these pulls are possible if there were only 12 red cubes, 13 green cubes, and 14 blue cubes in the sack.

    Part 1 Solution

    Part 1 again begins with importing the regex library. From there, I defined an upper limit for each color cube in a dictionary - 12 red, 13 green, and 14 blue as noted in the challenge. Each line was then divided using the "enumerate" function to keep a running count of each round. Using regex, color labels were found within each line, the values of which were identified by indexing the line in relation to each pull and color. If the number of cubes in each pull were less than our dictionary definitions, the ID for each game being tracked by the "enumerate" function was extracted and added to an empty "counts" variable. Once complete, the final "counts" variable is the answer to part one.

    
	import re
	def cube_counter(document):
	cube_limits = {'red': 12, 'green': 13, 'blue': 14}
	count = 0

	for id, game in enumerate(document.strip().split('\n'), start = 1):
	for num, color in re.findall(r'(\d+) (red|blue|green)', game):
		if cube_limits[color] < int(num):
		break
	else:
		count += id
	return count

The values of each color in each row need to be multiplied together, then each of those values need added together to give the correct answer.

    Day 3 begins with your arrival at a gondola lift station. This gondola will take you to the source of where the water for Snow Island should be. The problem is that the lift is not moving. The Elf engineer working on restoring the gondola to operating condition informs you that a part is missing from the engine responsible for powering the lift, but no one can identify which part is missing. You will need to search the engine schematic to identify the missing part.

    Each part in the schematic is marked as a symbol with numbers adjacent to the symbols denoting part numbers. By calculating the sum of the part numbers, you will be able to determine which part is missing.

    Part 1 Solution

    Part 1 starts with defining row and column lengths. This is important so that sybols, and in turn, part numbers, can be indexed and extracted from the document. From there a function is defined with parameters set to run while working with information within the length of the rows and columns, ignore all spacer symbols which do not count as a part symbol ("."), note the index of these symbols, then search the surrounding indexes for part numbers. These numbers are then added to a running total, which is the answer to part 1.

These gear ratios will then need to be added together to identify which gear is incorrect.

    The same principle of searching each row from part 1 applied, but this time, only "*" symbols with two numbers adjacent to them were identified. The symbol was indexed, the characters in the eight surrounding indexed locations were searched, and if two separate numbers were identified, they were multiplied together with this value being added to the running total, resulting in the answer to part two.

    Challenge Overview

    Link to the Day 4 challenge.

    After correctly locating the incorrect gear in Day 3, you take the gondola up to a warmer, humid, less snowy location known as Island Island. You ask the greeting Elf about the water source for Snow Island. Unfortunately, this elf is only the gondola operator and does not know about the water source. The operator proposes a deal that if you can help determine if they have won on some scratchcards they were gifted, you can take their boat to see the gardener.

    The Elf shows you a pile of scratchcards, each with a list of winning numbers, a divider, and a list of the player's numbers. The idea is to identify any player numbers that match any of the winning numbers. The first instance of a matching number results in one point. Any match after that doubles the point value of the card. You will need to find the total value of the Elf's cards.

    Part 1 Solution

    Regex will used in the Day 4 solutions, so installing the library is the first order of business. From there, each line can be stripped of its text and split, indexing the first set of numbers (the "winning" numbers) and the second set of numbers (the Elf's numbers). Going number by number in the Elf's numbers, they are compared to each number in the winning numbers. If one is located, a point is added to a count variable. The total value of the card can be found by raising two to the power of (count - 1). For example, if only one matching number was identified, two would be raised to (1 - 1), or zero, reuslting in a single point. Two matching would be two raised to (2 - 1), 1, resulting in a value of 2, and so on.

    The total value for each card is then added together, resulting in the answer to part 1 of Day 4.

    
	import re
	document = open("drive/MyDrive/Advent of Code 2023/2023day4.txt", "r").read()
	
	lines = document.strip().split('\n')
	total_winnings = 0
	
	for game in lines:
	  game = re.sub(r'^.*:\s*', '', str(game))
	  game_pieces = re.split('\s+', str(game))
	
	  winning_numbers = list(map(int, game_pieces[0:10]))
	  elf_numbers = list(map(int, game_pieces[11:]))
	
	  count = 0
	  for number in elf_numbers:
		if number in winning_numbers:
		  count += 1
	
	  if count > 0:
		total_winnings += 2**(count - 1)

    Part 2 Solution

    In order to correct this issue, the same regex will be used to strip, split, and index the values in each line, only this time a total number of scratchcrads will be defined and winnings will be indexed to scratchcards. While there are still winning numbers, you will continue to return scratchcards based on the points sytem in part 1 until the points come to zero.

    A running total of scratchcards is kept until no more are won and, when finished, the final number of scratchcards won is the answer to the part 2.

    Part 1 Solution

    In order to identify the source, destination, and associated values, the tables of numbers needed to be split into individual tables. From there, a regex was used to identify the various sets of numbers. These values were indexed, differences between the values were calculated, those values were mapped to each seed's associated indexes in the following tables, and planting location values were returned. From there, the lowest value was identified and marked as the starting point for planting operations.

    
	import re
	document = open("drive/MyDrive/Advent of Code 2023/2023day5.txt", "r").read()
	
	maps = document.split('\n\n')
	seeds = re.findall(r'\d+', maps[0])
	
	min_location = float('inf')
	for x in map(int, seeds):
		for seg in maps[1:]:
			for conversion in re.findall(r'(\d+) (\d+) (\d+)', seg):
				destination, start, delta = map(int, conversion)
				if x in range(start, start + delta):
					x += destination - start
					break
	
		min_location = min(x, min_location)
	
	print(f"Part 1: {min_location}")

    Part 2 Solution

    In part 2, you re-read the almanac and note that each sead noted is actually a range of seeds. That is, the first numbers listed are the identifiable values for each seeed/characteristic, while the second and fourth values are the length of the range of those values (74, 3, 19, 2 would equate to 74, 75, 76 and 19, 20, respectively). A new lowest value based on this information is required.

    To account for this, the code from part 1 was modified with to split each line into sets of numbers, with the ranges of the first number in each set being returned. This amended code accounts for any overlapping values and returns the lowest value within each range of seeds. The lowest of these values is the answer to part 2.

    
	import re
	document = open("drive/MyDrive/Advent of Code 2023/2023day5.txt", "r").read()
	
	maps = document.split('\n\n')
	intervals = []
	
	for seed in re.findall(r'(\d+) (\d+)', maps[0]):
		x1, dx = map(int, seed)
		x2 = x1 + dx
		intervals.append((x1, x2, 1))
	
	min_location = float('inf')
	while intervals:
		x1, x2, level = intervals.pop()
		if level == 8:
			min_location = min(x1, min_location)
			continue
	
		for conversion in re.findall(r'(\d+) (\d+) (\d+)', maps[level]):
			z, y1, dy = map(int, conversion)
			y2 = y1 + dy
			diff = z - y1
			#no overlap
			if x2 <= y1 or y2 <= x1:
				continue
			#split original interval at y1
			if x1 < y1:
				intervals.append((x1, y1, level))
				x1 = y1
			#split original interval at y2
			if y2 < x2:
				intervals.append((y2, x2, level))
				x2 = y2
			#perfect overlap -> make conversion and let pass to next
			intervals.append((x1 + diff, x2 + diff, level + 1))
			break
	
		else:
			intervals.append((x1, x2, level + 1))
	
	print(f"Part 2: {min_location}")

    Day 7

    Challenge Overview

    Link to the Day 7 challenge.

    The all-expenses-paid trip to Desert Island turns out to be a one-way trip. Upon arrival, an Elf inquires about whether you had brought the parts to fix the sand issues. They tell you to get on their camel with them to ride to the location of the issue. This journey will take a few days, so the Elf teaches you a card game called "Camel Cards". This game is similar to five-card poker, except there are no suits and there is a possibility to get five of a kind. Five of a kind beats four of a kind, which beats a full house, which beats three of a kind, which beats two pair, which beats a pair, which beats high card. If two hands have the same type, you compare each card in order to see which is higher. Once a card in that hand is higher than the other, the hand with the higher value card wins.

    To play, you are given a list of hands and their corresponding bid. You will need to develop a way to check and rank each hand, assign a ranking value to each, and multiply the bid by the ranking value. You then need to determine the total winnings for your set of hands.

    Part 1 Solution

    In order to check and rank each card, a few tools from the numpy, collections, and functools libraries need to be imported. This allowed me to count and sort the long list of hands. I started by allocating a value to each hand type and checking each individual hand in my input against it. Five of a kind returned a value of 5, four of a kind a value of 4, and so on.

    From there, once each hand had a value, they could be sorted. If more than one hand had the same value, each card was evaluated starting at the begining and ranked each individual card as a nested ranking order. Once these were sorted, they were placed back into the overall ranking sytem and the program continued. Once the hands were listed from strongest to weakest, their bids were multipled by their ordered place, and the sum of all multiplied values was recorded as the final answer to part 1.

    
	import numpy as np
	from collections import Counter, defaultdict
	from functools import cmp_to_key
	document = open("drive/MyDrive/Advent of Code 2023/2023day7.txt", "r").read()
	
	lines = document.strip().split("\n")
	
	def get_type(hand):
	  counts = defaultdict(int)
	  for card in hand:
		counts[card] += 1
	
	  amounts = sorted(counts.values())
	  if amounts == [5]:
		return 5
	  if amounts == [1, 4]:
		return 4
	  if amounts == [2, 3]:
		return 3.5
	  if amounts == [1, 1, 3]:
		return 3
	  if amounts == [1, 2, 2]:
		return 2.5
	  if amounts == [1, 1, 1, 2]:
		return 2
	  return 1
	
	labels = "AKQJT98765432"
	
	def compare(a, b):
	  #a and b are two hands
	  rankA = (get_type(a), a)
	  rankB = (get_type(b), b)
	  if rankA[0] == rankB[0]:
		if a == b:
		  return 0
		for i, j in zip(a, b):
		  if labels.index(i) < labels.index(j):
			return 1
		  if labels.index(i) > labels.index(j):
			return -1
		return -1
	  if rankA[0] > rankB[0]:
		return 1
	  return -1
	
	
	line_list = []
	for line in lines:
	  line = line.split()
	  line_list.append((line[0], int(line[1])))
	
	line_list = sorted(line_list, key=cmp_to_key(lambda x, y: compare(x[0], y[0])))
	ans = 0
	for i, line in enumerate(line_list):
	  ans += (i + 1) * line[1]
	
	print(ans)

    Part 2 Solution

    To make things interesting, the Elf lets you know that now "J" cards are jokers and that these can be considered wildcards. They can be used to match any card that gives the best hand, but when used for tie-breakers, they count less than the lowest value card (a "2"). With these new rules, the totals needed to be recalculated.

    The ranking system used previously would still work, but while passing the input through the function, "J"-value cards would need to be addressed. Once the cards are sorted by order, if a "J" was present, the card with the most value in the individual hand would be matched by the "J". The function ranked the cards as it did in the first part, but this time would adjust the value for "J" accordingly in tie-breakers. Sorting, bid multiplication, and total calculations were all the same once sorted and ranked to give me the answer for part 2.

    
	import numpy as np
	from collections import Counter, defaultdict
	from functools import cmp_to_key
	document = open("drive/MyDrive/Advent of Code 2023/2023day7.txt", "r").read()
	
	lines = document.strip().split("\n")
	
	
	def get_type(hand):
	  counts = defaultdict(int)
	  jokers = 0
	  for card in hand:
		if card == "J":
		  jokers += 1
		else:
		  counts[card] += 1
	
	  amounts = sorted(counts.values())
	  if jokers >= 5 or amounts[-1] + jokers >= 5:
		return 5
	  if jokers >= 4 or amounts[-1] + jokers >= 4:
		return 4
	
	  #full house
	  if amounts[-1] + jokers >= 3:
		rem_jokers = amounts[-1] + jokers - 3
		if len(amounts) >= 2 and amounts[-2] + rem_jokers >= 2 or rem_jokers >= 2:
		  return 3.5
		return 3
	
	  if amounts[-1] + jokers >= 2:
		rem_jokers = amounts[-1] + jokers - 2
		if len(amounts) >= 2 and amounts[-2] + rem_jokers >= 2 or rem_jokers >= 2:
		  return 2.5
		return 2
	
	  return 1
	
	labels = "AKQT98765432J"
	
	def compare(a, b):
	  #a and b are two hands
	  rankA = (get_type(a), a)
	  rankB = (get_type(b), b)
	  if rankA[0] == rankB[0]:
		if a == b:
		  return 0
		for i, j in zip(a, b):
		  if labels.index(i) < labels.index(j):
			return 1
		  if labels.index(i) > labels.index(j):
			return -1
		return -1
	  if rankA[0] > rankB[0]:
		return 1
	  return -1
	
	
	lines_list = []
	for line in lines:
	  line = line.split()
	  lines_list.append((line[0], int(line[1])))
	
	
	lines_list = sorted(lines_list, key=cmp_to_key(lambda x, y: compare(x[0], y[0])))
